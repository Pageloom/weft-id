# Sessions

Configure how long user sessions last and how WeftId handles inactive accounts.

Navigate to **Settings > Security > Sessions**.

## Maximum session length

How long a user can remain signed in before they must re-authenticate.

| Option |
|--------|
| Indefinitely (no time limit) |
| 8 hours |
| 1 day |
| 2 days |
| 3 days |
| 4 days |
| 5 days |
| 6 days |
| 1 week |
| 2 weeks |

## Persistent sessions

When enabled, sessions survive browser restarts. When disabled, users must sign in again each time they open their browser.

## Automatic inactivation

Automatically inactivate users who haven't been active for a set period. Once inactivated, an admin must reactivate them before they can sign in again.

| Option |
|--------|
| Disabled |
| After 14 days |
| After 30 days |
| After 60 days |
| After 90 days |

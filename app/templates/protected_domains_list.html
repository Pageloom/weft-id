Protected Domains

Register a DNS domain you want to protect with forward auth (for example acme-corp.com), so WeftID can gate HTTP apps behind it. You prove control of the domain with a DNS-TXT record, and point a portal host (e.g. auth.acme-corp.com) at WeftID. This is separate from privileged (email) domains, which route user identity by email address and do not protect web apps.
